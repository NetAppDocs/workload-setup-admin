= Custom dashboards overview
:icons: font
:imagesdir: ./media/

[.lead]
As a cloud and storage admin, custom dashboards give you a clear view of the systems and workloads you manage. They let you build and personalize views that match your work, priorities, and business goals, instead of being limited to preset layouts.

== Learn about custom dashboards

Custom dashboards use AI to create personalized views of your Workload Factory environment. You simply describe what you want to monitor in plain language, and the system automatically builds visual charts based on your needs.

=== Performance and access

Creating a dashboard usually takes more than 3 seconds, and a progress indicator appears during this time. After it is created, the dashboard opens in under 3 seconds. Each widget loads separately to help the dashboard run faster.

Everyone in your Workload Factory account can view all custom dashboards. This shared setup helps teams collaborate and see the same operational information across your organization.

=== Permissions

To create or view custom dashboards, you need the correct AWS IAM permissions for the resources you want to monitor. Make sure the Workload Factory account has read access to the data sources used by the dashboards.

link:add-credentials.html[Learn how to add AWS credentials and set up IAM permissions^].

=== Read-only design

Custom dashboards are read-only. They cannot change your storage or workload settings. This lets you monitor your system without accidentally changing anything.

=== Limitations

The following limitations currently apply to custom dashboards:

* You cannot share dashboards outside your Workload Factory account
* Only one user can edit a dashboard at a time
* Concurrent editing is not supported

=== Demo environment

If you're using a Workload Factory demo account, some prebuilt dashboards are available. They help you see what custom dashboards can do. Each dashboard shows a common monitoring need: disaster recovery status, capacity and costs across regions, and whether storage protection rules are being followed.

== Why use custom dashboards

Workload Factory users manage many workloads, environments, and priorities, but standard dashboards only show the vendor's preset views. Custom dashboards help solve several operational challenges:

* *Answer key questions quickly*: Focus on the most important questions or gaps in your understanding instead of general system statistics.
* *Monitor what matters now*: Track the metrics that match your current priorities, not a fixed set of health indicators.
* *Link technical data to business results*: Show how system performance affects operations and business goals.
* *Avoid switching between tools*: See all the relevant information in one place instead of jumping between different tools, views, or exports.

== What you can do with custom dashboards

The AI-guided dialog helps you define:

* Workloads and resources to monitor  
* Key metrics that matter for your work  
* Time ranges and periods to compare  
* Views by region, workload, or other categories  
* Rules or requirements you need to track and show

After creation, you can:

* Refresh dashboard data to see the latest information  
* Edit dashboards to change what they show  
* Delete dashboards you no longer need  
* View dashboard details, including who created it and when it was last updated
